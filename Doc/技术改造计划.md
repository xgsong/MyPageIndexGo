# PageIndex Go 版本技术改造计划 - 详细实现版

## 一、项目背景
PageIndex 是基于 LLM 的无向量、基于推理的 RAG 系统，当前 Go 版本已经实现了核心功能，但在处理大规模文档和高并发场景下仍存在性能瓶颈。本计划针对现有性能问题进行系统性优化，提升系统的处理效率、内存利用率和可扩展性。

## 二、优化目标
1. **端到端处理速度提升**：索引生成速度提升 30%-200%，搜索延迟降低 20%-50%
2. **内存使用优化**：内存占用降低 40%-60%，支持 GB 级超大文档处理
3. **系统吞吐量提升**：并发处理能力提升 2-5 倍
4. **API 成本降低**：LLM API 调用次数减少 30%-70%
5. **兼容性保证**：所有优化保持现有 CLI 和 API 接口不变，不破坏现有功能

#### 1. LLM 调用缓存机制
**具体实现方案**：
在 LLM Client 层实现基于文本哈希的响应缓存，使用 sync.Map 做内存缓存，相同输入直接返回缓存结果。

**关键代码片段**：
```go
// pkg/llm/client.go
type CachedLLMClient struct {
    llmClient LLMClient
    cache     *sync.Map // key: hash of input, value: cache entry
}

type CacheEntry struct {
    value     interface{}
    timestamp time.Time
}

func NewCachedLLMClient(client LLMClient) LLMClient {
    return &CachedLLMClient{
        llmClient: client,
        cache:     &sync.Map{},
    }
}

func (c *CachedLLMClient) GenerateStructure(ctx context.Context, text string) (*document.Node, error) {
    key := hashText("GenerateStructure", text)
    if entry, ok := c.cache.Load(key); ok {
        return entry.(*document.Node), nil
    }

    node, err := c.llmClient.GenerateStructure(ctx, text)
    if err != nil {
        return nil, err
    }

    c.cache.Store(key, node)
    return node, nil
}

func hashText(prefix, text string) string {
    h := sha256.New()
    h.Write([]byte(prefix))
    h.Write([]byte(text))
    return hex.EncodeToString(h.Sum(nil))
}
```

**技术选型对比**：
- 使用 sync.Map：原生支持并发读写，性能优秀，无需额外依赖
- 替代方案：github.com/golang/groupcache - 更复杂，适合分布式场景
- 决策依据：当前需求是单机内存缓存，sync.Map 足够满足性能要求

**代码修改点**：
- 在 `pkg/llm/` 目录下创建新文件 `cached_client.go`
- 修改 `NewOpenAIClient` 函数，添加是否启用缓存的参数
- 在 `indexer/generator.go` 中添加对缓存客户端的支持

**潜在风险和应对措施**：
- 内存泄漏风险：添加缓存过期机制，定期清理过期条目
- 缓存一致性风险：对于同一文档的不同版本，可能返回旧结果，需要添加缓存失效机制

**测试方案和性能验收指标**：

        MaxRetries:        5,
        BaseDelay:         time.Second,

// 在 OpenAI 客户端中识别 Retry-After 头
func (c *OpenAIClient) createChatCompletion(ctx context.Context, req openai.ChatCompletionRequest) (string, error) {
    var content string
    err := utils.DoRetry(ctx, utils.DefaultRetryConfig(), func() error {
        resp, err := c.client.CreateChatCompletion(ctx, req)
        if err != nil {
            // 检查是否是可重试错误
            var apiErr *openai.APIError
            if errors.As(err, &apiErr) {
                switch apiErr.HTTPStatusCode {
                case 429, 500, 503: // 限流、服务器错误

                    if retryAfter := getRetryAfter(err); retryAfter > 0 {
                        time.Sleep(retryAfter)
                    }
                    return err // 继续重试
                default:
                    return utils.StopRetry(err) // 不可重试的错误
                }
            }
            return err
        }
        if len(resp.Choices) == 0 {
            return fmt.Errorf("no choices returned from OpenAI")
        }
        content = resp.Choices[0].Message.Content
        return nil
    })
    return content, err
}

func getRetryAfter(err error) time.Duration {
    // 解析错误响应中的 Retry-After 头
    return 0
}

#### 3. 节点 ID 哈希索引
**具体实现方案**：
索引树生成时预先生成 `nodeID → *Node` 的哈希映射表，搜索时直接通过 ID 查找节点，替代递归遍历。

**关键代码片段**：
```go
// pkg/document/tree.go
type IndexTree struct {
    Root         *Node     `json:"root"`
    TotalPages   int       `json:"total_pages"`
    DocumentInfo string    `json:"document_info"`
    GeneratedAt  time.Time `json:"generated_at"`
    nodeMap      map[string]*Node `json:"-"` // 新增：ID 到节点的映射，不序列化
}

func NewIndexTree(root *Node, totalPages int) *IndexTree {
    tree := &IndexTree{
        Root:        root,
        TotalPages:  totalPages,
        GeneratedAt: time.Now(),
        nodeMap:     make(map[string]*Node),
    }
    tree.buildNodeMap() // 预构建节点映射
    return tree
}

func (t *IndexTree) buildNodeMap() {
    var traverse func(*Node)
    traverse = func(node *Node) {
        if node == nil {
            return
        }
        t.nodeMap[node.ID] = node
        for _, child := range node.Children {
            traverse(child)
        }
    }
    traverse(t.Root)
}

func (t *IndexTree) FindNodeByID(id string) *Node {
    return t.nodeMap[id]
}

// 在 llm/openai.go 中替换 findNodesByID 函数
func findNodesByID(tree *document.IndexTree, ids []string) []*document.Node {
    var result []*document.Node
    for _, id := range ids {
        if node := tree.FindNodeByID(id); node != nil {
            result = append(result, node)
        }
    }
    return result
}
```

**技术选型对比**：
- 使用 map[string]*Node：简单高效，O(1) 查找时间
- 替代方案：无，这是最直接的实现方式

**代码修改点**：
- 修改 `pkg/document/tree.go` 中的 `IndexTree` 结构体
- 在 `NewIndexTree` 函数中添加构建节点映射的逻辑
- 添加 `FindNodeByID` 方法
- 修改 `pkg/llm/openai.go` 中的 `findNodesByID` 函数

**潜在风险和应对措施**：

#### 4. 修复 PDF 字体缓存泄漏
**具体实现方案**：
在 `extractTextLayer` 函数结束后清理全局字体缓存，或者改为每个 PDF 实例独立缓存。

**关键代码片段**：
```go
// pkg/document/pdf.go
func (p *PDFParser) extractTextLayer(buf []byte) ([]Page, error) {
    reader := bytes.NewReader(buf)
    pdfReader, err := pdf.NewReader(reader, int64(len(buf)))
    if err != nil {
        return nil, fmt.Errorf("failed to open PDF for text extraction: %w", err)
    }

    numPages := pdfReader.NumPage()
    pages := make([]Page, 0, numPages)

    fonts := make(map[string]*pdf.Font) // 每个 PDF 实例独立缓存
    for i := 1; i <= numPages; i++ {
        page := pdfReader.Page(i)
        if page.V.IsNull() {
            pages = append(pages, Page{Number: i, Text: ""})
            continue
        }

        text, err := page.GetPlainText(fonts)
        if err != nil {
            return nil, fmt.Errorf("failed to extract text from page %d: %w", i, err)
        }

        pages = append(pages, Page{
            Number: i,
            Text:   text,
        })
    }

    // 不再需要显式清理，函数结束后会自动回收
    return pages, nil
}
```

**技术选型对比**：
- 每个实例独立缓存：最简单且有效的方案
- 替代方案：在函数结束后清理全局缓存
- 决策依据：每个 PDF 实例独立缓存避免了线程安全问题

**代码修改点**：
- 修改 `pkg/document/pdf.go` 中的 `extractTextLayer` 函数
- 确保 `fonts` 变量在函数内部声明，而不是使用全局变量

**潜在风险和应对措施**：
- 内存峰值风险：每个 PDF 解析会创建新的字体缓存，但函数结束后会自动回收
- 性能影响：重复解析同一字体的 PDF 会重新解析字体，但这种场景较少

**测试方案和性能验收指标**：
- 测试场景：长时间运行 PDF 解析任务，监控内存占用

#### 5. 动态并发控制
**具体实现方案**：
每次 API 调用后记录返回的 `X-RateLimit-Remaining` 和 `X-RateLimit-Reset` 头，用令牌桶算法动态调整并发数。

**关键代码片段**：
```go
// pkg/indexer/rate_limiter.go
type DynamicRateLimiter struct {
    maxConcurrency int
    minConcurrency int
    bucket         *rate.Limiter
    mutex          sync.Mutex
}

func NewDynamicRateLimiter(initialConcurrency, minConcurrency, maxConcurrency int) *DynamicRateLimiter {
    return &DynamicRateLimiter{
        maxConcurrency: maxConcurrency,
        minConcurrency: minConcurrency,
        bucket:         rate.NewLimiter(rate.Limit(initialConcurrency), initialConcurrency),
    }
}

func (r *DynamicRateLimiter) AdjustRate(remaining int, reset time.Time) {
    r.mutex.Lock()
    defer r.mutex.Unlock()

    // 根据剩余配额调整并发数
    newRate := calculateNewRate(remaining, reset)
    r.bucket.SetLimit(rate.Limit(newRate))
}

// 在 OpenAI 客户端中解析响应头
func (c *OpenAIClient) createChatCompletion(ctx context.Context, req openai.ChatCompletionRequest) (string, error) {
    var content string
    err := utils.DoRetry(ctx, utils.DefaultRetryConfig(), func() error {
        resp, err := c.client.CreateChatCompletion(ctx, req)
        if err != nil {
            return err
        }

        // 解析响应头
        remaining := getRateLimitRemaining(resp)
        reset := getRateLimitReset(resp)
        if dynamicLimiter != nil {
            dynamicLimiter.AdjustRate(remaining, reset)
        }

        if len(resp.Choices) == 0 {
            return fmt.Errorf("no choices returned from OpenAI")
        }
        content = resp.Choices[0].Message.Content
        return nil
    })
    return content, err
}
```

**技术选型对比**：

**代码修改点**：

func (g *IndexGenerator) generateAllSummaries(ctx context.Context, root *document.Node) error {
    // 收集需要生成摘要的节点
    var nodesToProcess []*document.Node
    var collect func(*document.Node)
    collect = func(node *document.Node) {
        if node == nil {
            return
        }
        if node.Summary == "" {
            nodesToProcess = append(nodesToProcess, node)
        }
        for _, child := range node.Children {
            collect(child)
        }

                if text, ok := g.pageTextMap[pageNum]; ok {
                    nodeText.WriteString(text)
                    nodeText.WriteString("\n\n")
                }
            }

            if nodeText.Len() > 0 {
                if err := g.GenerateSummariesForNode(ctx, node, nodeText.String()); err != nil {
                    return err
                }
            }
            return nil
        })
    }

#### 9. 批量 LLM 调用支持
**具体实现方案**：
将多个摘要生成请求合并为一个批量 API 调用，利用 LLM 的函数调用能力同时处理多个节点的摘要生成。

**关键代码片段**：
```go
// pkg/llm/openai.go
type BatchSummaryRequest struct {
    NodeID    string `json:"node_id"`
    Title     string `json:"title"`
    Text      string `json:"text"`
}

type BatchSummaryResponse struct {
    NodeID   string `json:"node_id"`
    Summary  string `json:"summary"`
    Error    string `json:"error"`
}

func (c *OpenAIClient) GenerateBatchSummaries(ctx context.Context, requests []*BatchSummaryRequest) ([]*BatchSummaryResponse, error) {
    prompt := batchSummaryPrompt()

    req := openai.ChatCompletionRequest{
        Model: c.model,
        Messages: []openai.ChatCompletionMessage{
            {
                Role:    openai.ChatMessageRoleUser,
                Content: prompt + fmt.Sprintf("%v", requests),
            },
        },
        Functions: []openai.FunctionDefinition{
            {
                Name:        "generate_summaries",
                Description: "Generate summaries for multiple document nodes",
                Parameters: &struct {
                    Type       string              `json:"type"`
                    Properties map[string]interface{} `json:"properties"`
                    Required   []string            `json:"required"`
                }{
                    Type: "object",
                    Properties: map[string]interface{}{
                        "summaries": map[string]interface{}{
                            "type": "array",
                            "items": map[string]interface{}{
                                "type": "object",
                                "properties": map[string]interface{}{
                                    "node_id": map[string]interface{}{"type": "string"},
                                    "summary": map[string]interface{}{"type": "string"},
                                },
                                "required": []string{"node_id", "summary"},
                            },
                        },
                    },
                    Required: []string{"summaries"},
                },
            },
        },
        FunctionCall: &openai.FunctionCall{Name: "generate_summaries"},
    }

    content, err := c.createChatCompletion(ctx, req)
    if err != nil {
        return nil, err
    }

    // 解析函数调用结果
    var response struct {
        Summaries []*BatchSummaryResponse `json:"summaries"`
    }
    if err := json.Unmarshal([]byte(content), &response); err != nil {
        return nil, err
    }
    return response.Summaries, nil
}
```

**技术选型对比**：

- 决策依据：函数调用更灵活，不需要等待批处理作业完成

**代码修改点**：
- 修改 `pkg/llm/openai.go` 以支持批量调用

- 修改 `indexer/generator.go` 以使用批量调用

#### 11. 索引树序列化优化
**具体实现方案**：

**关键代码片段**：
```go
// pkg/output/json.go
import (
    "os"
    "github.com/goccy/go-json" // 替换 encoding/json
)

func WriteIndexTreeToFile(tree *document.IndexTree, filename string) error {
    file, err := os.Create(filename)
    if err != nil {
        return err
    }
    defer file.Close()

    // 使用高性能 JSON 库
    encoder := json.NewEncoder(file)
    encoder.SetIndent("", "  ")
    err = encoder.Encode(tree)
    if err != nil {
        return err
    }
    return nil
}

func LoadIndexTreeFromFile(filename string) (*document.IndexTree, error) {
    file, err := os.Open(filename)
    if err != nil {
        return nil, err
    }
    defer file.Close()

    var tree document.IndexTree
    err = json.NewDecoder(file).Decode(&tree)
    if err != nil {
        return nil, err
    }

    // 重新构建节点映射（如果使用了节点 ID 哈希索引）
    tree.buildNodeMap()

    return &tree, nil
}
```

func (g *IndexGenerator) Update(ctx context.Context, existingTree *document.IndexTree, newDoc *document.Document) (*document.IndexTree, error) {
    // 生成新文档的索引
    newTree, err := g.Generate(ctx, newDoc)
    if err != nil {
        return nil, err
    }

    // 合并索引树
    mergedTree, err := mergeIndexTrees(existingTree, newTree)
    if err != nil {
        return nil, err
    }

    return mergedTree, nil
}

func mergeIndexTrees(existing, new *document.IndexTree) (*document.IndexTree, error) {
    // 合并逻辑：
    // 1. 比较根节点的内容
    // 2. 合并相同标题的章节
    // 3. 处理冲突的内容
    // 4. 添加新章节

    // 简单示例：直接合并根节点的子节点
    mergedRoot := document.NewNode("Merged Document", 1, existing.TotalPages+new.TotalPages)
    mergedRoot.AddChild(existing.Root)
    mergedRoot.AddChild(new.Root)

    mergedTree := document.NewIndexTree(mergedRoot, existing.TotalPages+new.TotalPages)
    mergedTree.DocumentInfo = fmt.Sprintf("Merged document with %d pages", mergedTree.TotalPages)

}
